아코디언

아래 예제 섹션은 아코디언 디자인 패턴을 실례로 보여주는 3개 섹션으로 나뉘어진 간단한 개인 정보 입력 양식을 포함하고 있습니다. 이 구현에서, 한 개 아코디언 패널이 항상 확장 되어 있으며, 한 번에 오직 하나의 패널만 확장 될 수 있습니다.

예

접근성 기능

시각적 설계에는 사용자가 아코디언이 향상된 키보드 탐색 기능을 제공한다는 것을 이해하도록 돕기 위한 기능이 포함됩니다. 아코디언 헤더 버튼에 키보드 초점이 있을 경우, 아코디언 컨테이너와 모든 헤더 버튼의 스타일링이 변경됩니다.

아코디언 헤더 버튼이 초점을 받는 경우:

  • 전체 아코디언을 둘러싸는 테두리의 색상이 변경됩니다.
  • 아코디언 헤더 버튼의 배경색이 변경됩니다.

초점을 받은 아코디언 헤더 버튼:

  • 헤더 텍스트와 아이콘을 모두 포함하는 테두리를 가지게 합니다.
  • 초점을 얻지 않은 다른 아코디언과 구별되는 배경색을 가지게 합니다.

키보드 지원

키 기능
Space 또는 Enter 축소된 섹션의 아코디언 헤더에 초점이 있는 경우, 섹션을 확장시킵니다.
Tab
  • 초점을 다음 초점을 얻을 수 있는 요소(focusable element)로 이동시킵니다.
  • 아코디언의 모든 초점을 얻을 수 있는 요소(focusable element)는 페이지 Tab 순서에 포함됩니다.
Shift + Tab
  • 초점을 이전 초점을 얻을 수 있는 요소(focusable element)로 이동시킵니다.
  • 아코디언의 모든 초점을 얻을 수 있는 요소(focusable element)는 페이지 Tab 순서에 포함됩니다.
Down Arrow
  • 초점이 아코디언 헤더에 있는 경우, 초점을 다음 아코디언 헤더로 이동시킵니다.
  • 초점이 마지막 아코디언 헤더이 있는 경우, 초점을 첫 번째 아코디언 헤더로 이동시킵니다.
Up Arrow
  • 초점이 아코디언 헤더에 있는 경우, 초점을 이전 아코디언 헤더로 이동시킵니다.
  • 초점이 첫 번째 아코디언 헤더에 있는 경우, 초점을 마지막 아코디언 헤더로 이동시킵니다.
Home 초점이 아코디언 헤더에 있는 경우, 초점을 첫 번째 아코디언 헤더로 이동시킵니다.
End 초점이 아코디언 헤더에 있는 경우, 초점을 마지막 아코디언 헤더로 이동시킵니다.

역할(role), 속성(property), 상태(state), Tabindex 속성(attribute)

역할(role) 속성(attribute) 요소(element) 사용법
h3
  • 아코디언 헤더로 제공되는 요소(element)
  • 각 아코디언 헤더 요소(element)는 그것의 콘텐트 패널의 가시성을 제어하는 버튼을 포함합니다.
  • 예제는 제목 수준 3을 사용하므로 페이지의 개요에 정확히 들어맞습니다; 이 예는 수준 2의 제목이 있는 섹션에 포함됩니다.
aria-expanded="true" button 아코디언 패널이 확장되는 경우 true로 설정하고, 그렇지 않으면false로 설정.
aria-controls="ID" button 헤더가 제어하는 패널의 ID를 가리킵니다.
aria-disabled="true" button 아코디언 패널이 확장되고 축소가 허용되지 않는 다면, true로 설정합니다.
region div 현재 확장된 아코디언 패널을 포함하는 랜드마크 영역을 생성합니다.
aria-labelledby="IDREF" div
  • region 요소(element)에 대한 접근 가능한 이름을 정의합니다.
  • 영역을 확장하고 축소하는 아코데이너 헤더 버튼을 참조합니다.
  • region 요소(element)는 랜드마크로 식별될 수 있는 접근 가능한 이름을 가지도록 요구됩니다.
